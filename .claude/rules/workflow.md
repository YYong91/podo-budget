# 작업 워크플로우

## 태스크 관리 (TASK.md)

PM 에이전트가 `TASK.md`(루트)에 작업을 정의하고 담당 에이전트(기획, BE, FE)에게 할당한다.

- PM이 작업 분배 및 상태 관리를 담당
- 각 서브에이전트는 자기 섹션의 작업을 확인하고 수행
- 작업 시작 시 `[~]`, 완료 시 `[x]`로 상태 업데이트
- 완료된 작업은 하단 "완료된 작업" 섹션으로 이동
- TASK.md는 .gitignore에 포함 (로컬 전용)

## 커밋 전 체크

커밋 전에 반드시 아래 사항을 확인한다:

1. **ruff 린트/포맷** — `ruff check --fix backend/` → `ruff format backend/`
2. **시크릿 스캔** — .env, API 키, SSH 키, 패스워드가 커밋에 포함되지 않는지 확인
3. **테스트 통과** — `pytest` 실행하여 기존 테스트 깨지지 않는지 확인
4. pre-commit 훅이 설치되어 있으면 자동으로 실행되지만, 수동 확인도 습관화

## 마무리 프로토콜
작업이 끝날 때마다 아래 파일들에 업데이트할 내용이 있는지 체크하고 반영한다:

1. **CLAUDE.md** — 프로젝트 구조, 명령어, 아키텍처 변경사항
2. **.claude/rules/*.md** — 새로 확립된 컨벤션이나 규칙
3. **MEMORY.md** (`~/.claude/projects/` 하위) — 세션 간 기억할 패턴, 디버깅 인사이트, 사용자 선호
